第一章 Accustoming Yourself to Objective-C

第一章 你需要熟悉Object-C

Objective-C(之后简称OC)是把面向对象(object-oriented)的特性引入到C中的一门全新语言。OC语言经常被描述为冗长的，因为它使用的方法名极其的长并且使用了很多方括号还有没有空隙。这个特性导致OC源代码可读性非常好但是对于使用C++或者Java语言开发的人员来说经常会感觉困难。 编写OC语言代码能学得很快但是也有很多错综复杂的需要清楚知道的特性经常会被忽略了。同样的，有一些特性被滥用或者一知半解的，这样产生的代码的维护或者调试是很困难的。这一章节的内容涵盖了OC语言的基本要点；接下来的章节涵盖了OC语言的具体内容和相关的框架。

第一节：你需要熟悉Object-C的根

OC语言跟其他面向对象的语言比如C++和Java是相似的，但是也在很多方面有不同的。如果你有其他面向对象语言的使用经验，你将会理解许多用过的范式和模式。但是跟其他面向对象语言相比，OC语言的语法更多的接近外星语言，因为OC语言使用的是一种消息传递结构而不是方法回调。OC语言从一种使用消息来源方式的Smalltalk语言进化而来。消息传递和方法回调这两者的不同看起来像这样：
//Messaging (Object-C)
Object *obj =new Object; [obj performWith:parameterl and:parameter2];

//Function calling(C++)
Object *obj = new Object;
obj -> perform(parameterl, parameter2);
这其中关键的不同点是消息传递结构的代码是由OC的运行时决定代码如何得到执行。而方法回调的代码是由其编译器决定代码将会如何被执行。这个方法回调的例子里面介绍了多态性(polymorphism)，表现为其运行时查找的一种形式是通过使用著名的虚拟表来进行的。但是消息传递的查找总是在OC的运行时中。实际上，OC的编译器不在乎关于这个对象的类型被传递过来没有。这个查找也在OC的运行时中,通过一种著名的叫动态绑定的过程，关于动态绑定更多详细的信息在第11节。
OC语言是运行时组件做了很多重要的任务而不是编译器。OC的运行时包含了所有的使OC语言去工作所需要的关于面向对象特性的数据结构和方法。例如，OC的运行时包括了所有的内存管理(memory-management)方法。基本上)(Essentially)，OC的运行时设置代码是像胶水一样把你所有的代码粘合在一起，然后你的代码和来自于以一种动态库的形式的东西链接起来。因此(Thus)，每当OC的运行时更新时，你的应用可以从性能提高里面获得好处。一种大多数任务是在编译时工作的语言需要重新编译才能获得类似性能提高之类的好处。
