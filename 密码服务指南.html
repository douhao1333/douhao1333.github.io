##概述 OS X 和 iOS提供的一些密码服务包括： * 加密和解密（包括通用的和特殊的） * 使用钥匙串的密钥管理 * 加密强随机数生成 * 安全通讯（SSL和TLS） * 使用FileVault和iOS文件的安全存储 ###**加密、签名和验证还有数字证书能保护数据免受窥视** 有两个主要的加密类型：对称加密，是一种使用单个共享密钥对数据加密和解密的方法；和非对称加密，是用一个密钥对数据加密和另一个分离(但是相关)密钥对数据解密。你能使用哈希来检测对数据片段的修改。你可以使用哈希和非对称密钥结合生成一个数字签名，当根据数据证书进行验证时，证明数据片段的来源。数字证书，反过来，通过验证签署证书的的一方的签名进行验证，然后验证该方的证书，直到得到一个你所信任的固定的证书，这个证书被称为`锚点证书(anchor certificate)`。 相关章节: `深度密码学概念`([Cryptography Concepts In Depth](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/cryptoservices/CryptographyConcepts/CryptographyConcepts.html#//apple_ref/doc/uid/TP40011172-CH8-SW1))
###**OS和iOS提供加密和散列API** OS和iOS提供了一些列加密和散列数据的API，包括`钥匙串服务(Keychain Services)`; `密码消息语法服务(Cryptographic Message Syntax Services)`; `证书(Certificate)`, `密钥(Key)`,`和证书服务(Trust Services)`; `通用密码(Common Crypto)`; `安全转换(Security Transforms)`。 相关章节: `加密和散列数据`([Encrypting and Hashing Data ](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/cryptoservices/GeneralPurposeCrypto/GeneralPurposeCrypto.html#//apple_ref/doc/uid/TP40011172-CH9-SW1))
###**钥匙串帮助你存储密码信息** 如果你的应用程序必须存储加密密钥，密码，证书和其他安全相关的信息。都可以使用钥匙串。钥匙串为小信息提供安全存储，因此运行在系统上的其他应用无法访问，只有在用户登录或者解锁设备之后才能访问。OS X 和 iOS提供了两套API用于对钥匙串的操作和从钥匙串中获取密钥：证书，密钥，信任服务API和钥匙串服务API。 相关章节: `管理密钥、证书和密码`([Managing Keys, Certificates, and Passwords](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/cryptoservices/KeyManagementAPIs/KeyManagementAPIs.html#//apple_ref/doc/uid/TP40011172-CH11-SW1))
###**OS X 和 iOS提供密码安全随机数生成** 一些加密任务需要你去生成一些密码性强的伪随机数。OS X 通过`/dev/random`设备节点来提供这些数字。iOS则通过随机化服务API来提供这些数字。 相关章节: `生成随机数`([Generating Random Numbers ](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/cryptoservices/RandomNumberGenerationAPIs/RandomNumberGenerationAPIs.html#//apple_ref/doc/uid/TP40011172-CH12-SW1))
###**OS X 和 iOS提供安全网络通讯API** 数据传输安全需要一个安全通讯渠道。OS X 和 iOS提供了一系列API用于建立安全通讯渠道，包括网址加载系统，在`Core Foundation`和`Foundation`的套接字流，和安全传输。 相关章节: `数据传输安全`([Transmitting Data Securely ](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/cryptoservices/SecureNetworkCommunicationAPIs/SecureNetworkCommunicationAPIs.html#//apple_ref/doc/uid/TP40011172-CH13-SW1))
###**已经弃用的技术** 尽管`CDSA`和`CSSM`API在OS X 10.7及以后的版本都被弃用了，但你可能在有些情况下需要使用它。基于这种原因，这个文档提供了一个附录。 相关章节: `CDSA 概述`([CDSA Overview ](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/cryptoservices/CDSA/CDSA.html#//apple_ref/doc/uid/TP40011172-CH4-SW1))
##先决条件 在阅读这个文档之前，你应该熟悉了`安全概述`([Security Overview](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/Security_Overview/Introduction/Introduction.html#//apple_ref/doc/uid/TP30000976))和`安全编码指南`([Secure Coding Guide](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/SecureCodingGuide/Introduction.html#//apple_ref/doc/uid/TP40002415))。 ##也可以看看 有关OS X身份验证和授权（基于加密技术构建的）的详细信息，请阅读`身份验证，授权和权限指南`([Authentication, Authorization, and Permissions Guide](https://developer.apple.com/library/prerelease/content/documentation/Security/Conceptual/AuthenticationAndAuthorizationGuide/Introduction/Introduction.html#//apple_ref/doc/uid/TP40011200))。

OS X 和 iOS提供了一系列技术用于支撑密码服务——加密和解密，散列，随机数生成，安全网络通讯等等。这些技术能被用来保护静态数据(当存储在你的硬盘驱动器或者其他媒体上)，保护传输的数据，确定第三方的身份，构建额外的安全技术。


概述

OS X 和 iOS提供的一些密码服务包括：

  • 加密和解密（包括通用的和特殊的）
  • 使用钥匙串的密钥管理
  • 加密强随机数生成

  • 使用FileVault和iOS文件的安全存储
