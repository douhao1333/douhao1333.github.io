##什么是加密？ 加密是将数据转换为不使用一些密钥的就无法获取的形式。类似的被转换的数据被称为`密文(ciphertext)`。使用密钥来反转这个过程并将数据返回到其原始(`明文(cleartext)`或`纯文本(plaintext)`)形式被称为`解密(decryption)`。OS X 和 iOS的大多数安全API在某种程序上都依赖于文本或者数据的加密。例如，加密被用于证书和数字签名的创建，用于安全存储钥匙串中的秘密，和用于信息的安全传输。 加密是任何从一个字符串替换成另一个字符串的简单过程，这个过程中密钥是替换规则，替换规则是一个复杂的数学算法。以安全为目的，解密密文越困难，越好。另一方面，如果算法太复杂，会花费很长的时间加密或者需要的密钥太大无法轻松存储，对于使用个人电脑的情况这种做法是不切实际的。因此，在加密的强度(就是某人发现算法和密钥有多困难)和易用性之间达到一个平衡。 ##加密的类型 在电脑安全中有两种主要的加密类型，名称为`对称密钥加密(symmetric key encryption)`和`非对称密钥加密(asymmetric key encryption)`。在与加密密切相关的过程中，使用密钥和不可能被逆转的数学算法来转换数据，被称作加密散列。本节的其余部分讨论加密密钥，密钥交换机制(包括使用在某些安全传输协议中的Diffie-Hellman密钥交换)和加密哈希函数。 ###**对称密钥** `对称密钥加密(Symmetric key cryptography)`(也称为`秘密密钥加密( secret key cryptography)`)是大多数人熟悉的密钥的经典使用：同一个密钥用于加密和解密数据。这个经典的，最容易破解的，是凯撒密码(命名为`Julius Caesar`)这个版本，它将消息中的每个字母替换为字母表中固定数量位置的字母(例如，`a`被替换为`c`，`b`被替换为`d`等等)。在凯撒密码中，用于加密和解密数据的密钥仅仅是字母表旋转位置的数量和旋转的方向。现在对称加密算法更复杂和更难以破解。但是，使用同一个密钥来加密和解密数据这个属性仍是同样的。 对称密钥加密有许多不同的算法可以使用，提供从最简约的到近乎难以破解的安全任何算法。一些算法提供强大的安全性，在代码上可以简单的扩展，和快速加解密。这样的算法对于加密存储在计算机上的文件在未授权的个人去访问这个电脑的时候保护电脑的这一目的是很有用的。但是对于从一台电脑发送消息到另一台电脑它不太有用，因为在通讯通道的两端必须拥有密钥并且保证它的安全。这样的密钥的分发和安全存储是困难的并且有可能打开安全漏洞。 在1968年，美国海军的情报舰`USS Pueblo`被朝鲜捕获。那时，每艘海军船只在各种安全级别上为各种代码机提供对称密钥。每个密钥每天都更换。因为`USS Pueblo`情报舰被朝鲜拥有但不知道有多少密钥被`Pueblo`的船员销毁，所以海军假设`USS Pueblo`情报舰上的所有的密钥都被攻破了。在太平洋战区的每艘船和后勤站(那是成千上万的设备，包括在海里的船只)通过他们携带的码本和穿孔卡片去物理替换每台设备的所有密钥。 这个`Pueblo`事件是一个极端情况。然而，它与在互联网上为商业提供安全通讯的问题有共同点。在这两种情况下，代码用于发送安全信息，不是在两个位置，而是在服务器(互联网服务器或者海军通讯中心)和无数通讯者(个人用户或者船只和后勤站)之间。参与安全通讯的最终用户越多，分发和保护这个秘密对称密钥的问题就越大。 尽管交换和创造对称密钥的技术在一些程度上能克服这个问题(例如，`Diffie-Hellman`密钥交换，在后面的章节会了解到)，但一个更实用的解决方案是使用为了电脑通讯而产生的实用算法发明叫做非对称密钥解密。 ###**非对称密钥** 非对称密钥加密，加密和解密数据的密钥是不同的。 非对称密钥算法最有用的是一个密钥不可能从另一个密钥推测出来。这种情况下，一个密钥是公开的而另一个密钥保持安全。这种技术通常被称为是`公共密钥加密(public key cyptography)`，并且提供了一些超过对称加密的明显的优势：消除了分发安全密钥给大量用户得必要性，并且该算法能被用于授权以及加密。 在1977年由`Ron Rivest`，` Adi Shamir`和 `Len Adleman`描述的第一个公钥算法已经广泛可用，从他们的名字缩写该算法被称为RSA加密。尽管从此之后其他的公钥算法被创建，但是RSA仍然是最通用的。RSA的数学方法超出了本文档的范围，可以在互联网和很多密码学书籍里面找到。该算法基于两个大素数和它们的乘积的数学操作。该算法的强度被认为与一大个数的因子分解的困难有关。使用现代数字电脑的当前和可预见的速度，在RSA密钥的生成中选择足够长的素数使这个算法无限期的安全。但是，这种信念在数学上没有被证实，并且可能是一种快速分解算法或者一种完全不同的方式去破解RSA加密。还有，如果实用量子计算机被发展出来，分解大的数字将不会是棘手的问题。 图 1 - 1所示的安全信息交换有以下几个步骤： * 1.`Alice`使用公钥算法之一来生成了一对加密密钥：`Alice`她保密的私钥，和一个公钥。她还准备发送信息给`Bob`。 * 2.`Alice`发送了一个未加密的公钥给`Bob`。因为她的私钥不能从公钥推测出来，这样做不会以任何方式损害她的私钥。 * 3.`Alice`现在能轻松的向`Bob`证明她的身份(一个被称为`身份认证(authentication)`的过程)。这样做，她现在使用她的私钥去加密她的信息(或者信息的任何一部分)然后把信息发送给`Bob`。 * 4.`Bob`使用`Alice`的公钥去解密这个信息。这可以证明消息必须来自`Alice`，因为只有她有这个用于加密这个信息的私钥。 * 5.`Bob`使用`Alice`的公钥去加密他的信息然后把信息发送给`Alice`。这个消息是安全的，因为即使它被截获，除了`Alice`没有人有解密它所需的私钥。 * 6.`Alice`使用她的私钥去解密这消息。 由于加密和认证在国家安全和保护公司秘密方面是非常有趣的主题，一些非常聪明的人致力于创建安全系统并尝试去破解它。因此，不要惊讶实际安全通讯和认证程序比前面描述的更复杂的情况。例如，使用你的私钥加密消息的认证方法可以被`中间人攻击(man-in-the-middle attack)`绕过，一些怀有恶意意图的人(在密码学书籍中通常被称为`Eve`)截获`Alice`的源消息然后使用她们的信息代替源消息，因此`Bob`没有使用`Alice`的公钥，而是使用`Eve`的公钥。`Eve`截获`Alice`的每条消息，使用`Alice`的公钥解密消息，篡改消息(如果`Eve`她愿意)，然后使用她的私钥重新加密消息。当`Bob`收到消息，他用`Eve`的公钥来解密消息，但是他以为公钥是来自`Alice`的。 尽管这是一个太宽泛和技术性的主题但是在本文档的后面会详细介绍，数字证书和数字签名也可以解决这些安全问题。这些技术会在后面的章节中去了解。 ##Diffie-Hellman密钥交换 这个`Diffie-Hellman 密钥交换(key exchange)`协议是一种在不安全信道上的通讯会话的两端去生成共享对称密钥的方式。`Diffie-Hellman`技术通常使用跟RSA公钥加密技术相似的数学方法实现。但是，也可以使用跟椭圆曲线加密相似的技术。它的基本步骤如下： * 1.`Alice`和`Bob`互相交换公钥。 * 对于RSA，密钥必须有相同的模数部分,`p`。 * 对于椭圆曲线加密，用于加密的域参数必须是商定好的。 按照这个规则，你使用的模数和域参数的值应该是`RFC 5114`指定的。 * 2.`Alice`和`Bob`分别使用他们的私钥加密一个共享非秘密的值`g`，然后他们互相交换这个加密值。 `g`的值一般也来自`RFC 5114`，但是当使用RSA时选择另一个值，`g`的值必须是原始根对`p`求余，即是任何一个跟`p`除了1以外没有相同除数的数，该数与`g`对`p`求余的幂相等。 * 3.`Alice`使用她的私钥对从`Bob`收到的加密值进行加密，反之亦然。这个值被用作共享会话密钥。 由此，尽管双方都不知道另一方的私钥，但是双方的会话密钥是相同的。第三方截取公钥但是缺乏双方任何一方的私钥无法生成一个会话密钥。因此，这个数据加密使会话密钥在中转的时候是安全的。 尽管`Diffie-Hellman`密钥交换提供了强大的保护抵御破解去截获数据。但是它没有提供确认在连接的另一端的实体是你认为的人这个机制。就是说，这个协议容易遭到中间人攻击。因此，它有时跟一些别的认证技术一起使用确保数据受信任。

  • 5.Bob使用Alice的公钥去加密他的信息然后把信息发送给Alice。这个消息是安全的，因为即使它被截获，除了Alice没有人有解密它所需的私钥。
  • 6.Alice使用她的私钥去解密这消息。
    由于加密和认证在国家安全和保护公司秘密方面是非常有趣的主题，一些非常聪明的人致力于创建安全系统并尝试去破解它。因此，不要惊讶实际安全通讯和认证程序比前面描述的更复杂的情况。例如，使用你的私钥加密消息的认证方法可以被中间人攻击(man-in-the-middle attack)绕过，一些怀有恶意意图的人(在密码学书籍中通常被称为Eve)截获Alice的源消息然后使用她们的信息代替源消息，因此Bob没有使用Alice的公钥，而是使用Eve的公钥。Eve截获Alice的每条消息，使用Alice的公钥解密消息，篡改消息(如果Eve她愿意)，然后使用她的私钥重新加密消息。当Bob收到消息，他用Eve的公钥来解密消息，但是他以为公钥是来自Alice的。
    尽管这是一个太宽泛和技术性的主题但是在本文档的后面会详细介绍，数字证书和数字签名也可以解决这些安全问题。这些技术会在后面的章节中去了解。

Diffie-Hellman密钥交换

这个Diffie-Hellman 密钥交换(key exchange)协议是一种在不安全信道上的通讯会话的两端去生成共享对称密钥的方式。Diffie-Hellman技术通常使用跟RSA公钥加密技术相似的数学方法实现。但是，也可以使用跟椭圆曲线加密相似的技术。它的基本步骤如下：

  • 1.Alice和Bob互相交换公钥。
    • 对于RSA，密钥必须有相同的模数部分,p。
    • 对于椭圆曲线加密，用于加密的域参数必须是商定好的。
      按照这个规则，你使用的模数和域参数的值应该是RFC 5114指定的。
  • 2.Alice和Bob分别使用他们的私钥加密一个共享非秘密的值g，然后他们互相交换这个加密值。
    g的值一般也来自RFC 5114，但是当使用RSA时选择另一个值，g的值必须是原始根对p求余，即是任何一个跟p除了1以外没有相同除数的数，该数与g对p求余的幂相等。
  • 3.Alice使用她的私钥对从Bob收到的加密值进行加密，反之亦然。这个值被用作共享会话密钥。

由此，尽管双方都不知道另一方的私钥，但是双方的会话密钥是相同的。第三方截取公钥但是缺乏双方任何一方的私钥无法生成一个会话密钥。因此，这个数据加密使会话密钥在中转的时候是安全的。
尽管Diffie-Hellman密钥交换提供了强大的保护抵御破解去截获数据。但是它没有提供确认在连接的另一端的实体是你认为的人这个机制。就是说，这个协议容易遭到中间人攻击。因此，它有时跟一些别的认证技术一起使用确保数据受信任。
